Responsive Email XX

I started coding responsive emails in early 2013, but after creating a few templates I noticed that I'd reinvented the wheel each time. So I created a master template with a few layout ideas that adapt to small screens using media queries. I also included workarounds for common HTML email gotchas such as dodgy box model support in Outlook, auto-detected links in Yahoo!, and background images in Gmail.

With most of my best-practices in one template, I creating a solid starting point for new emails. I simply ripped out what I didn't need and customized the rest.

This was too awesome to keep to myself, so I named it Responsive Email XX (mostly because I was drinking Dos Equis while writing it), and put it up on GitHub.

Cerberus, Another Responsive Email

Responsive Email XX (above) was my first deep dive with responsive email. I used it for a while and it had some success on GitHub, but after a while its flaws began to surface.

So I built another one that combines the best of what's in Responsive Email XX with what I've learned since. I wrote a blog post explaining why I made this template, how it works, and what I learned from my second deep dive with responsive emails.

This template, called Cerberus, focuses on a small number of responsive email patterns that, when combined, go a long way. A mantra of email design is to keep things simple. Where my first attempt failed, I hope this one succeeds.

Support Details

When Canfield's tech support team starts helping a client, conversations usually start with “What is your O.S. / browser / network settings”? Seeing an opportunity to work on something fun, I designed an app that automatically grabs the technical details of a computer environment and routes them via email.

Using this design, a co-worker and I built a working prototype in JavaScript on our spare time. The resulting app is quite portable and has already been forked into Canfield projects. Not a bad way to spend our free time, eh?
