Setting the Bar for
HTML Emails

One of my primary responsibilities was coding and deploying HTML email designs. My team shipped about 10-15 in a normal week, and I was the point person in my group for emails.

I frequently experimented with ways to improve each email. Simple things like adding image dimensions and background colors so the emails looked good before images downloaded. And creating subtle features like drop shadows in code, without images.

Ultimately, I created a best practices guide for our group.

Speed Counts

In my down time, I led the charge in improving Toys “R” Us's website performance (page speed).

Using tools like Web Page Test, GT Metrix, YSlow and Google PageSpeed, I helped gather data for key pages and present the results to the brass.

We eventually made front-end optimization part of our process, with our biggest win coming with the Toys “R” Us and Babies “R” Us home pages. We removed over half the payload, removed half of the requests, and replaced bulky, inaccessible flash files with JavaScript.

Creative Code

Due to a less-than-trustful content management system, I was often forced to be creative with my code to implement designs. Basic things like CSS resets and JavaScript weren't allowed by the system, so we worked with the graphic designers to find workable compromises. Along the way, I digitized a print-centric style-guide using some creative font stacks.
